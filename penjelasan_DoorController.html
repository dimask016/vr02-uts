Analisis Detail Skrip Pengendali Pergerakan Pintu Ganda (UTS_DoorController.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip UTS_DoorController.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman logika kalkulasi posisi dinamis pintu kiri dan kanan, sistem interaksi tombol, serta interpolasi pergerakan sliding ganda di Unity.

1. Library/Namespace Imports

using UnityEngine;
using UnityEngine.UI;

Mengimpor pustaka (library) eksternal yang dibutuhkan oleh skrip:

  • using UnityEngine; : Mengakses core engine Unity untuk mengenali komponen mekanis seperti Transform, Vector3, dan fungsi pergerakan fisik.
  • using UnityEngine.UI; : Digunakan agar skrip mengenali komponen Button dan Image untuk antarmuka interaksi tombol pintu.

2. Deklarasi Variabel & Komponen Pintu

public class UTS_DoorController : MonoBehaviour
{
    [Header ("Door Button")]
    [SerializeField] Button _doorButton;
    private Image buttonImage;

    [Header ("Left Door")]
    [SerializeField] Transform _leftDoor;
    private Vector3 leftDoorDefaultPosition;
    private Vector3 leftDoorTargetPosition;

    [Header ("Right Door")]
    [SerializeField] Transform _rightDoor;
    private Vector3 rightDoorDefaultPosition;
    private Vector3 rightDoorTargetPosition;

Mendeklarasikan properti fisik dan referensi objek pintu:

  • _doorButton & buttonImage : Tombol UI yang menempel di panel dinding serta penampung komponen visual gambarnya untuk diubah warnanya secara dinamis.
  • _leftDoor & _rightDoor : Mengambil data koordinat 3D (Transform) dari sayap pintu kiri dan sayap pintu kanan.
  • ...DefaultPosition : Menyimpan koordinat awal (posisi menutup rapat) saat game pertama kali dijalankan.
  • ...TargetPosition : Menyimpan koordinat tujuan akhir (posisi terbuka lebar) hasil kalkulasi otomatis.

3. Identifikasi GameController & ID Pintu

    [Header ("GameController")]
    [SerializeField] UTS_GameController _gameController;
    public string Door;
  • _gameController : Menghubungkan skrip ini ke pusat logika permainan agar dapat saling bertukar informasi status.
  • public string Door; : Pengenal unik berbentuk teks string (contoh diisi: "security", "office", atau "control") di Inspector untuk menentukan identitas pintu ini.

4. Kalkulasi Otomatis Arah Sliding Pintu (Fungsi Start)

    void Start()
    {
        buttonImage = GetComponent<Image>();
        _doorButton.onClick.AddListener(OnDoorButtonClick);
        leftDoorDefaultPosition = _leftDoor.position;
        rightDoorDefaultPosition = _rightDoor.position;

        if(_leftDoor.localScale.x > _leftDoor.localScale.z)
        {
            leftDoorTargetPosition = new Vector3(_leftDoor.position.x - 5f, _leftDoor.position.y, _leftDoor.position.z);
        }
        else
        {
            leftDoorTargetPosition = new Vector3(_leftDoor.position.x, _leftDoor.position.y, _leftDoor.position.z + 5f);
        }
        // ... (Kalkulasi yang sama berlaku untuk _rightDoor dengan arah berlawanan)
    }

Inisialisasi cerdas saat game dimulai agar satu skrip bisa dipakai untuk pintu yang menghadap ke mana saja:

  • Mendapatkan komponen gambar tombol dan mendaftarkan fungsi klik tombol (AddListener).
  • Mengunci posisi awal pintu ke dalam variabel DefaultPosition.
  • if(_leftDoor.localScale.x > _leftDoor.localScale.z) : Sistem mendeteksi orientasi rotasi pintu di map. Jika skala X lebih besar dari Z, pintu dianggap menghadap horizontal, sehingga saat terbuka pintu kiri akan digeser mundur sejauh 5 unit pada sumbu X (x - 5f). Jika sebaliknya, pintu digeser 5 unit pada sumbu Z (z + 5f).

5. Loop Deteksi Transisi Pintu (Fungsi Update)

    void Update()
    {
        ChangeButtonColor();
        if(_gameController.GetDoorStatus(Door))
        {
            AnimateDoorOpen();
        }
        else
        {
            AnimateDoorClose();
        }
    }

Mengeksekusi logika visual di setiap frame:

  • ChangeButtonColor(); : Terus memperbarui warna lampu tombol indikator sesuai hak akses terbaru pemain.
  • _gameController.GetDoorStatus(Door) : Meminta informasi status ke pusat data game controller berdasarkan ID string pintu saat ini. Jika bernilai true (terbuka), maka fungsi pergerakan membuka diaktifkan; jika false (tertutup), fungsi pergerakan menutup dijalankan.

6. Trigger Interaksi Klik Tombol (OnDoorButtonClick)

    public void OnDoorButtonClick()
    {
        _gameController.ToggleIsOpen(Door);
    }
  • Fungsi ini dipicu sesaat setelah pemain menekan tombol fisik UI di layar. Kode langsung meneruskan perintah dengan memanggil fungsi ToggleIsOpen(Door) milik Game Controller untuk memvalidasi apakah pemain berhak membuka pintu tersebut atau memunculkan popup error jika syarat belum terpenuhi.

7. Interpolasi Animasi Translasi (AnimateDoorOpen & Close)

    public void AnimateDoorOpen()
    {
        _leftDoor.position = Vector3.MoveTowards(_leftDoor.position, leftDoorTargetPosition, 5f * Time.deltaTime);
        _rightDoor.position = Vector3.MoveTowards(_rightDoor.position, rightDoorTargetPosition, 5f * Time.deltaTime);
    }

    public void AnimateDoorClose()
    {
        _leftDoor.position = Vector3.MoveTowards(_leftDoor.position, leftDoorDefaultPosition, 5f * Time.deltaTime);
        _rightDoor.position = Vector3.MoveTowards(_rightDoor.position, rightDoorDefaultPosition, 5f * Time.deltaTime);
    }

Logika matematika pergerakan pergeseran objek pintu secara halus tanpa animasi bawaan Unity (Timeline/Animator):

  • Vector3.MoveTowards(...) : Fungsi kalkulasi linier yang menggeser posisi objek dari titik A ke titik B dengan kecepatan konstan yang presisi.
  • 5f * Time.deltaTime : Menentukan kecepatan sliding pintu (skala 5) yang dikalikan dengan waktu frame berjalan agar pergerakan pintu terlihat halus, mulus, dan seragam di semua komputer (tidak bergantung pada tinggi/rendahnya FPS perangkat).

8. Sinkronisasi Warna Indikator Tombol (ChangeButtonColor)

    public void ChangeButtonColor()
    {
        buttonImage.color = _gameController.GetDoorColor(Door);
    }
}
  • buttonImage.color = _gameController.GetDoorColor(Door); : Mengubah warna visual komponen tombol panel interaksi di dinding secara langsung. Skrip mengambil data warna dari fungsi GetDoorColor di GameController. Lampu tombol akan berubah menjadi hijau jika akses terbuka/diijinkan, dan merah jika akses ditolak/dikunci.
