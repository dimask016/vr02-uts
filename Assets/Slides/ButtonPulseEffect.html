🔄 ButtonPulseEffect.cs

✨ UI Effect · Runtime · 38 baris

Fungsi utama: Membuat tombol UI berdenyut (pulse) secara terus-menerus dengan skala naik-turun menggunakan Mathf.PingPong. Efek ini menarik perhatian pemain ke tombol interaktif, sangat berguna untuk menandai stasiun yang sedang aktif atau harus diklik.

📌 Baris Kode (cuplikan) 🎯 Fungsi / Metode 📖 Penjelasan
using UnityEngine;
using UnityEngine.UI;
using System.Collections;		 Namespace Mengimpor namespace yang diperlukan: UnityEngine untuk MonoBehaviour, UnityEngine.UI untuk Image, dan System.Collections untuk Coroutine (IEnumerator).
public float pulseSpeed = 2f;
public float minScale = 0.95f;
public float maxScale = 1.05f;		 Parameter Pulse pulseSpeed = kecepatan denyut (semakin tinggi semakin cepat). minScale dan maxScale adalah batas bawah dan atas skala saat berdenyut. Nilai default membuat tombol sedikit mengecil dan membesar.
private Image image;
private Vector3 originalScale;
private Color originalColor;		 Variabel Internal Menyimpan referensi komponen Image, skala asli tombol, dan warna asli (meskipun warna tidak digunakan dalam efek ini, disimpan untuk kemungkinan pengembangan lebih lanjut).
void Start()
{
image = GetComponent<Image>();
originalScale = transform.localScale;
if (image != null) originalColor = image.color;
StartCoroutine(Pulse());
}		 Start() Inisialisasi: mengambil komponen Image, menyimpan skala dan warna asli, lalu memulai coroutine Pulse() yang akan berjalan terus menerus selama objek aktif.
IEnumerator Pulse()
{
float t = 0f;
while (true)
{
t += Time.deltaTime * pulseSpeed;
float lerp = Mathf.PingPong(t, 1f);
float scale = Mathf.Lerp(minScale, maxScale, lerp);
transform.localScale = originalScale * scale;
yield return null;
}
}		 Pulse() – Coroutine Logika denyut:
1. t bertambah setiap frame berdasarkan pulseSpeed.
2. Mathf.PingPong(t, 1f) menghasilkan nilai osilasi antara 0 dan 1 (seperti gelombang segitiga).
3. Mathf.Lerp(minScale, maxScale, lerp) mengubah nilai osilasi menjadi skala antara minScale dan maxScale.
4. Skala tombol diperbarui setiap frame (yield return null).
Loop while (true) membuat efek berjalan terus menerus.
// Efek independen dari interaksi Karakteristik Berbeda dengan ButtonClickEffect yang responsif terhadap input, ButtonPulseEffect bekerja secara otomatis dan terus-menerus tanpa perlu interaksi pengguna. Cocok untuk menarik perhatian ke tombol yang penting.
// Dapat dikombinasikan dengan ButtonClickEffect Kombinasi Kedua efek (ClickEffect dan PulseEffect) dapat dipasang pada tombol yang sama. ButtonPulseEffect akan membuat tombol berdenyut, sementara ButtonClickEffect akan menambahkan efek hover/press di atasnya.
