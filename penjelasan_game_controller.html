Analisis Detail Skrip Pusat Logika Permainan (UTS_GameController.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip UTS_GameController.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman logika *State Machine*, pengelolaan UI TextMeshPro, serta sistem validasi pintu evakuasi di Unity.

1. Library/Namespace Imports

using UnityEngine;
using UnityEngine.UI;
using UnityEngine.SceneManagement;
using TMPro;
using System.Collections;

Mengimpor pustaka (library) eksternal yang dibutuhkan oleh skrip:

  • using UnityEngine; : Library dasar Unity untuk mengenali komponen seperti MonoBehaviour, GameObject, dan fungsi bawaan engine.
  • using UnityEngine.UI; : Digunakan untuk mengendalikan komponen UI klasik Unity seperti komponen Image.
  • using UnityEngine.SceneManagement; : Dibutuhkan untuk memanipulasi perpindahan atau pemuatan ulang level/scene game.
  • using TMPro; : Digunakan agar skrip mengenali komponen teks modern berkemampuan tinggi, yaitu TextMeshProUGUI.
  • using System.Collections; : Library standar C# yang wajib disertakan agar fitur jeda waktu (Coroutine / IEnumerator) bisa berjalan.

Eksekusi otomatis satu kali di awal game:

  • Memanggil fungsi sinkronisasi UI agar teks misi dan peta langsung menampilkan visual default yang sesuai.
  • if (_warningCanvasGroup != null) : Proteksi dari error *NullReferenceException* (jika lupa memasang objek di Inspector), dilanjutkan menyembunyikan panel peringatan (alpha = 0f).

6. Loop Validasi Game Over (Fungsi Update)

    void Update()
    {
        if(gameOver)
        {
            Time.timeScale = 0f;
            if(Input.GetKeyDown(KeyCode.Return) || Input.GetKeyDown(KeyCode.KeypadEnter))
            {
                RestartGame();
            }
        } 
    }

Berjalan terus menerus setiap frame permainan:

  • Mengecek jika variabel gameOver aktif, maka skala waktu internal game dihentikan total (Time.timeScale = 0f), membekukan seluruh simulasi fisik dan mekanik game.
  • Dalam keadaan beku tersebut, skrip mendeteksi jika pemain menekan tombol **Enter** utama (Return) atau Enter area numpad (KeypadEnter) untuk segera memicu fungsi muat ulang permainan.

7. Interaksi & Validasi Pembukaan Pintu (ToggleIsOpen)

    public void ToggleIsOpen(string door)
    {
        if(door == "corridor") corridorOpen = !corridorOpen;

        if(door == "security")
        {
            if(!hasKeyCard) ShowWarningPopup("Take ID Card First!", true);
            securityOpen = (hasKeyCard) ? !securityOpen : false;
        }

        if(door == "office") officeOpen = !officeOpen;

        if(door == "control")
        {
            if(!hasKeyCard) ShowWarningPopup("Take ID Card First!", true);
            if(!alarmActive) ShowWarningPopup("Activate Alarm First!", true);
            controlRoomOpen = (hasKeyCard && alarmActive) ? !controlRoomOpen : false;
        }

        if(door == "exit")
        {
            if(!hasKeyCard) ShowWarningPopup("Take ID Card First!", true);
            if(!alarmActive) ShowWarningPopup("Activate Alarm First!", true);
            if(!exitDoorActive) ShowWarningPopup("Activate Exit First!", true);
            exitOpen = (hasKeyCard && alarmActive && exitDoorActive) ? !exitOpen : false;
        }
        StartCoroutine(closeDoor(door));
    }

Fungsi publik utama untuk menerima perintah pembukaan pintu berdasarkan nama ID string parameter:

  • Pintu corridor dan office tidak memiliki prasyarat, status langsung dibalik (terbuka/tertutup) lewat logika tanda seru (!).
  • Pintu security mengecek status kepemilikan kartu. Pernyataan Ternary (hasKeyCard) ? !securityOpen : false berarti: jika kartu dipegang, balikkan status pintu, jika tidak ada kartu, paksa pintu tetap terkunci rapat (false).
  • Pintu control dan exit memberlakukan sistem pengecekan berantai berlapis. Jika salah satu quest melompati urutan, popup peringatan kegagalan khusus akan ditampilkan.
  • StartCoroutine(closeDoor(door)); : Di baris akhir, fungsi menyalakan sistem timer otomatis agar pintu yang baru terbuka bisa menutup sendiri beberapa saat kemudian.

8. Sistem Penutup Pintu Otomatis (Coroutine)

    IEnumerator closeDoor(string door)
    {
        yield return new WaitForSeconds(autoCloseDelay);
        if(door == "corridor") corridorOpen = false;
        if(door == "security") securityOpen = false;
        if(door == "office") officeOpen = false;
        if(door == "control") controlRoomOpen = false;
        if(door == "exit") exitOpen = false;
    }
  • yield return new WaitForSeconds(autoCloseDelay); : Baris instruksi jeda yang memerintahkan Unity untuk menunda pemrosesan baris berikutnya selama nilai variabel penentu (5 detik).
  • Baris selanjutnya mendeteksi nama pintu asal yang dipanggil, lalu secara paksa menyetel nilai booleannya kembali ke false agar animasi/status fisik pintu tertutup kembali secara otomatis demi keamanan area.

9. Fungsi Getter Status & Warna Indikator Pintu

    public bool GetDoorStatus(string door)
    {
        if(door == "corridor") return corridorOpen;
        if(door == "security") return securityOpen;
        if(door == "office") return officeOpen;
        if(door == "control") return controlRoomOpen;
        if(door == "exit") return exitOpen;
        return false;
    }

    public Color GetDoorColor(string door)
    {
        if(door == "corridor") return accessColor;
        if(door == "security") return (hasKeyCard) ? accessColor : deniedColor;
        if(door == "office") return accessColor;
        if(door == "control") return (hasKeyCard && alarmActive) ? accessColor : deniedColor;
        if(door == "exit") return (hasKeyCard && alarmActive && exitDoorActive) ? accessColor : deniedColor;
        return deniedColor;
    }

Dua fungsi publik penyedia informasi (*Getter*) untuk digunakan oleh skrip eksternal (seperti detektor animasi pintu):

  • GetDoorStatus : Mengembalikan kondisi fisik pintu saat ini (apakah bernilai true/buka atau false/tutup).
  • GetDoorColor : Mengembalikan data warna lampu pemandu panel pintu. Jika syarat kepemilikan item/objektif dipenuhi, ia mengirimkan warna hijau (accessColor), jika tidak terpenuhi ia mengirimkan warna merah (deniedColor).

10. Fungsi Setter Penyelesaian Objektif (State Modifiers)

    public void SetHasKeyCard()
    {
        hasKeyCard = true;
        ShowSuccessFeedback("ID Card Collected!");
        UpdateChecklistUI(); UpdateMapUI();
        StartCoroutine(UpdateStatusAfterDelay());
    }

    public void SetAlarmActive()
    {
        if(alarmActive) { ShowWarningPopup("Alarm Already Active!", true); return; }
        alarmActive = true;
        ShowSuccessFeedback("Alarm Activated!");
        UpdateChecklistUI(); UpdateMapUI();
        StartCoroutine(UpdateStatusAfterDelay());
    }

Kumpulan fungsi publik yang dipanggil oleh objek interaksi (seperti tombol/kartu kunci) saat dipicu pemain:

  • Mengubah data status internal bersangkutan menjadi true.
  • Mengeluarkan umpan balik suara/teks sukses melalui ShowSuccessFeedback.
  • Memperbarui sistem visual UI pendaftaran misi beserta titik koordinat warna peta mini secara real-time.
  • if(alarmActive) { ... return; } : Proteksi ganda (Guard Clause). Jika pemain menekan tombol alarm berkali-kali padahal sudah aktif, fungsi langsung dihentikan prematur (return) agar tidak membuang performa komputasi.
  • *Catatan: Fungsi SetEmergencyExitActive() memiliki struktur penulisan dan logika pengamanan yang identik dengan fungsi alarm di atas.

11. Manajemen Kemenangan & Game Over (SetEvacFinish)

    public void SetEvacFinish()
    {
        isEvacuationComplete = true;
        ShowSuccessFeedback("EVACUATION COMPLETE! You are safe.");
        UpdateChecklistUI(); UpdateMapUI();
        StartCoroutine(UpdateStatusAfterDelay());
        StartCoroutine(SetGameOver(true));
    }

    public IEnumerator SetGameOver(bool success)
    {
        ShowPopupRoutine("Play Again? (Press Enter)", false);
        if(success)
        {
            ShowSuccessFeedback("EVACUATION COMPLETE! You are safe.");
            _txtGameOver.color = new Color(0f, 1f, 0f, 1f);
        }
        else
        {
            ShowWarningPopup("EVACUATION FAILED!", true);
            _txtGameOver.color = new Color(1f, 0f, 0f, 1f);
        }
        yield return new WaitForSeconds(5f);
        gameOver = true;
    }

Logika ketika pemain berhasil menyentuh garis finish atau kehabisan waktu evakuasi:

  • SetEvacFinish() : Menandai evakuasi selesai, memperbarui UI, lalu memulai Coroutine akhir hayat game.
  • SetGameOver(bool success) : Mengambil keputusan visual akhir level berdasarkan parameter sukses:
    • Jika success == true, tulisan utama diubah menjadi Hijau murni (Menang).
    • Jika success == false, tulisan diubah menjadi Merah menyala (Gagal/Kalah).
  • yield return new WaitForSeconds(5f); : Memberikan jeda waktu dramatis selama 5 detik agar pemain bisa membaca status hasil akhir mereka di layar sebelum variabel kunci gameOver = true diaktifkan.

Mengelola perubahan string dinamis daftar objektif di panel layar utama game:

  • Jika variabel bernilai true, warna teks seketika diubah menjadi Hijau (colorDone) dan modifikasi visual tulisan diberikan efek coretan tengah (FontStyles.Strikethrough). Jika belum selesai, warna teks tetap kuning penanda aktif.

13. Sinkronisasi UI Peta Mini (UpdateMapUI)

    public void UpdateMapUI()
    {
        if (_mapIDCard != null) _mapIDCard.color = hasKeyCard ? colorDone : colorActive;
        if (_mapAlarm != null) _mapAlarm.color = alarmActive ? colorDone : (hasKeyCard ? colorActive : colorTodo);
        if (_mapDoor != null) _mapDoor.color = exitDoorActive ? colorDone : (alarmActive ? colorActive : colorTodo);
        if (_mapExit != null) _mapExit.color = isEvacuationComplete ? colorDone : (exitDoorActive ? colorActive : colorTodo);
    }

Mengatur warna ikon di peta mini untuk mengarahkan rute navigasi pemain secara tidak langsung:

  • Ikon penanda lokasi dicat menggunakan skema tiga warna: Abu-abu (colorTodo) jika petunjuk belum boleh dikerjakan, Kuning (colorActive) jika tempat itu merupakan target mutlak saat ini, dan Hijau (colorDone) jika tugas di ruangan tersebut telah sukses dilakukan.

14. Sistem Animasi Popup & Kilatan Layar (Feedback & Lerp)

    IEnumerator ShowPopupRoutine(string message, bool isError)
    {
        // ... Menyiapkan pesan teks ...
        float elapsed = 0f;
        while (elapsed < 0.3f)
        {
            _warningCanvasGroup.alpha = Mathf.Lerp(0f, 1f, elapsed / 0.3f);
            elapsed += Time.deltaTime;
            yield return null;
        }
        _warningCanvasGroup.alpha = 1f;
        yield return new WaitForSeconds(2.5f);
        // ... (Proses kebalikannya untuk animasi Fade-Out ke 0f) ...
    }

Membuat efek transisi kemunculan teks popup peringatan secara halus (*Fade-In* dan *Fade-Out*):

  • while (elapsed < 0.3f) : Loop matematika yang berjalan di setiap frame selama durasi 0.3 detik.
  • Mathf.Lerp(0f, 1f, elapsed / 0.3f) : Interpolasi linier untuk mengubah nilai transparansi secara bertahap dari 0 murni (tidak terlihat) menjadi 1 murni (terlihat padat sempurna).
  • elapsed += Time.deltaTime; : Menambahkan akumulasi waktu nyata dari perangkat komputer untuk memastikan kecepatan animasi popup berjalan konsisten di semua FPS komputer.
  • yield return null; : Memerintahkan Unity untuk menjeda baris kode di titik ini dan melanjutkannya kembali tepat di frame berikutnya.

15. Kamus Hint & Muat Ulang Game (RestartGame)

    string GetCurrentObjectiveHint()
    {
        if (!hasKeyCard) return "Go to Office and Take ID Card!";
        if (!alarmActive) return "Go to Security and Activate Alarm!";
        if (!exitDoorActive) return "Go to Control Room and Open Exit Door!";
        if (!isEvacuationComplete) return "Go to Exit !";
        return "🏆 Evacuation Complete!";
    }

    public void RestartGame()
    {
        Time.timeScale = 1f;
        SceneManager.LoadScene(SceneManager.GetActiveScene().name);
    }
  • GetCurrentObjectiveHint() : Mesin pengecek logika berjenjang. Fungsi ini bertugas mendeteksi status objektif, lalu mengembalikan untaian string kalimat petunjuk yang sesuai untuk diletakkan pada teks layar utama.
  • Time.timeScale = 1f; : **Sangat Penting!** Mengembalikan kecepatan waktu internal Unity yang sebelumnya dibekukan (0f) menjadi normal kembali (1f). Tanpa baris ini, level yang baru dimuat akan membeku total secara permanen.
  • SceneManager.LoadScene(...) : Mendapatkan nama string dari file level yang sedang aktif saat ini, kemudian langsung memuat ulang scene dari detik pertama demi memulai siklus game baru.
