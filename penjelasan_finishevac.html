Analisis Detail Skrip Zona Akhir Evakuasi (UTS_FinishEvac.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip UTS_FinishEvac.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman logika deteksi zona pemicu (Trigger Collider) yang menentukan kapan pemain dinyatakan berhasil mencapai titik aman dan memenangkan permainan.

1. Library/Namespace Imports

using UnityEngine;

Mengimpor pustaka dasar engine:

  • using UnityEngine; : Menyediakan fungsionalitas fundamental Unity agar skrip ini dapat mengenali kelas inti seperti MonoBehaviour, komponen Collider, sistem fisik, serta metode deteksi tabrakan.

2. Deklarasi Kelas & Variabel Komponen (Serialized Fields)

public class UTS_FinishEvac : MonoBehaviour
{
    [SerializeField] UTS_GameController _gameController;
    [SerializeField] BoxCollider _finishCollider;
  • public class UTS_FinishEvac : MonoBehaviour : Mendeklarasikan blueprint kelas bernama UTS_FinishEvac yang mewarisi sifat dasar Unity (MonoBehaviour) agar dapat dipasang ke GameObject di dalam hierarki scene.
  • Atribut [SerializeField] memaksa variabel privat agar terekspos di jendela Inspector Unity, mempermudah proses menghubungkan referensi objek:
    • _gameController : Menyimpan referensi ke komponen skrip pengontrol utama (UTS_GameController) untuk mengirimkan sinyal kemenangan.
    • _finishCollider : Menyimpan referensi komponen tabrakan berbentuk kotak (BoxCollider) yang bertindak sebagai sensor batas garis finish di area permainan.

    // Start is called once before the first execution of Update after the MonoBehaviour is created
    void Start()
    {

    }

    // Update is called once per frame
    void Update()

  • Start() dan Update() : Merupakan fungsi siklus hidup (lifecycle) bawaan yang digenerasikan otomatis oleh Unity. Pada skrip ini, kedua fungsi dibiarkan kosong karena area finish tidak memerlukan inisialisasi awal atau pengecekan logika berkala di setiap framenya. Pengecekan hanya dilakukan jika terjadi peristiwa/event tertentu saja.

4. Logika Deteksi Sensor Finish (OnTriggerEnter)

    public void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            _gameController.SetEvacFinish();
        }
    }
}

Fungsi pemicu (*callback event*) dari sistem fisika Unity yang otomatis berjalan ketika ada objek lain menembus area BoxCollider sensor (dengan catatan opsi *Is Trigger* pada collider tersebut telah diaktifkan):

  • Collider other : Parameter yang menampung informasi data collider milik objek asing yang baru saja masuk atau menembus ke dalam area sensor finish.
  • if (other.CompareTag("Player")) : Melakukan validasi penyaringan menggunakan sistem Tag. Struktur ini memastikan bahwa fungsi di dalamnya hanya akan berjalan jika objek yang menembus garis finish tersebut adalah karakter pemain (Player), bukan objek lain seperti musuh, barang jatuh, atau peluru.
  • _gameController.SetEvacFinish(); : Ketika dipastikan bahwa objek tersebut adalah Player, skrip ini langsung memanggil fungsi SetEvacFinish() yang berada di dalam master controller untuk menghentikan timer, memicu UI kemenangan, dan mengubah state game menjadi selesai.
