Analisis Detail Skrip Sakelar Alarm Evakuasi (UTS_AlarmButton.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip UTS_AlarmButton.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman logika interaksi tombol alarm UI yang harus diaktifkan pemain sebagai syarat perkembangan misi evakuasi selanjutnya.

1. Library/Namespace Imports

using UnityEngine;
using UnityEngine.UI;

Mengimpor pustaka (library) bawaan Unity yang diperlukan oleh skrip ini:

  • using UnityEngine; : Library dasar utama Unity agar skrip mengenali kelas MonoBehaviour, komponen SerializeField, dan struktur data internal engine lainnya.
  • using UnityEngine.UI; : Mengijinkan skrip untuk mengakses, memantau, dan mengontrol komponen UI sistem klasik Unity, khususnya objek bertipe Button.

2. Deklarasi Kelas & Variabel Referensi Komponen

public class UTS_AlarmButton : MonoBehaviour
{
    [SerializeField] Button _alarmButton;
    [SerializeField] UTS_GameController _gameController;
  • public class UTS_AlarmButton : MonoBehaviour : Mendeklarasikan kelas utama bernama UTS_AlarmButton yang mewarisi sifat dari MonoBehaviour agar dapat disematkan ke dalam komponen objek di hierarki Unity.
  • Atribut [SerializeField] digunakan untuk mengekspos variabel privat ke jendela Inspector agar bisa dihubungkan secara visual:
    • _alarmButton : Menyimpan referensi komponen tombol UI (Button) fisik di layar yang akan ditekan pemain untuk menyalakan sirine.
    • _gameController : Menyimpan referensi ke skrip pusat pengatur alur permainan (UTS_GameController) untuk mengirimkan pemberitahuan perubahan state.

3. Registrasi Event Klik Tombol (Fungsi Start)

    // Start is called once before the first execution of Update after the MonoBehaviour is created
    void Start()
    {
        _alarmButton.onClick.AddListener(OnAlarmButtonClick);
    }

Fungsi siklus hidup Unity yang dieksekusi satu kali tepat sebelum frame pertama diperbarui:

  • _alarmButton.onClick.AddListener(OnAlarmButtonClick); : Berfungsi untuk mendaftarkan (*subscribe*) fungsi lokal bernama OnAlarmButtonClick ke dalam event klik tombol onClick.
  • Melalui baris ini, ketika pemain berinteraksi dan mengklik tombol alarm tersebut di antarmuka game, Unity akan langsung memicu jalannya fungsi OnAlarmButtonClick secara otomatis.

4. Fungsi Update Default

    // Update is called once per frame
    void Update()
    {

    }
  • Update() : Fungsi bawaan Unity yang dijalankan berulang kali di setiap frame. Pada skrip ini fungsi dibiarkan kosong tanpa kode karena sistem aktivasi alarm murni menggunakan arsitektur berbasis event (*Event-Driven*), sehingga menghemat beban pemrosesan CPU karena tidak perlu mengecek data apapun di setiap frame.

5. Eksekusi Logika Pusat Kebijakan (OnAlarmButtonClick)

    public void OnAlarmButtonClick()
    {
        _gameController.SetAlarmActive();
    }
}

Fungsi publik utama yang dipanggil sesaat setelah event klik terdeteksi:

  • _gameController.SetAlarmActive(); : Berfungsi untuk menginstruksikan skrip master pengontrol game agar menjalankan fungsi SetAlarmActive().
  • Di dalam master controller, pemanggilan fungsi ini akan mengubah status game menjadi *Alarm Active*, memicu kilatan visual hijau sukses di layar, memperbarui teks misi ke-2 (Activate Alarm) menjadi coret centang hijau, serta mengubah indikator warna lokasi target alarm pada mini-map.
