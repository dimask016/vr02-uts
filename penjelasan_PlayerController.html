Analisis Detail Skrip Penggerak & Interaksi Pemain (PlayerController.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip PlayerController.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman sistem kendali sudut pandang orang pertama (First-Person), sistem fisika pergerakan karakter, serta metode deteksi jarak interaksi tombol UI dan objek 3D menggunakan teknik Raycasting.

1. Namespace & Atribut Komponen Wajib

using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;
using UnityEngine.EventSystems;

[RequireComponent(typeof(CharacterController))]
public class PlayerController : MonoBehaviour

Memasukkan pustaka sistem dan pengaturan prasyarat komponen objek:

  • using UnityEngine.EventSystems; : Diperlukan agar karakter dapat mendeteksi dan mengirimkan event klik ke elemen UI Canvas di dalam dunia 3D.
  • [RequireComponent(typeof(CharacterController))] : Atribut pelindung yang memastikan GameObject tempat skrip ini menempel **wajib** memiliki komponen CharacterController. Jika belum ada, Unity secara otomatis akan menambahkannya untuk mencegah terjadinya error akibat komponen yang hilang.

2. Variabel Konfigurasi Sistem

    [Header("Pergerakan WASD")]
    public float moveSpeed = 5f;

    [Header("Kamera & Mouse Look")]
    public Camera playerCamera;
    public float mouseSensitivity = 2f;
    public float upperLookLimit = -80f;
    public float lowerLookLimit = 80f;

    [Header("Sistem Interaksi")]
    public float interactDistance = 3f;

    [Header("Crosshair UI Optional")]
    public Image crosshairImage;

Pengaturan parameter kontrol di dalam jendela Inspector Unity:

  • moveSpeed : Menentukan kecepatan translasi berjalan pemain berjalan.
  • mouseSensitivity : Skala sensitivitas pergerakan mouse saat memutar arah pandang kamera.
  • upperLookLimit & lowerLookLimit : Batas sudut rotasi vertikal (mendongak ke atas dan menunduk ke bawah) agar kamera tidak berputar terbalik melewati batas kepala.
  • interactDistance : Jarak jangkauan maksimal (misal: 3 meter) bagi pemain untuk dapat menekan tombol panel pintu atau mengambil benda.

3. Inisialisasi Kunci Kursor (Fungsi Start)

    void Start()
    {
        characterController = GetComponent<CharacterController>();
        if (playerCamera == null) playerCamera = Camera.main;

        Cursor.lockState = CursorLockMode.Locked;
        Cursor.visible = false;
        if (crosshairImage != null) crosshairImage.raycastTarget = false;
    }

Inisialisasi status sistem di awal permainan:

  • Cursor.lockState = CursorLockMode.Locked; : Mengunci pointer mouse agar tetap berada di titik tengah layar game.
  • Cursor.visible = false; : Menyembunyikan panah kursor mouse agar tidak mengganggu pandangan pemain selama bermain game FPS.
  • crosshairImage.raycastTarget = false; : Mematikan deteksi raycast pada gambar crosshair (titik tengah) agar tidak menghalangi laser deteksi dari kamera saat menembak objek di depannya.

4. Pemrosesan Logika Input Tiap Frame (Fungsi Update)

    void Update()
    {
        HandleRotation();
        HandleMovement();

        if (Input.GetMouseButtonDown(0) || Input.GetKeyDown(KeyCode.E))
        {
            TryInteractFromCrosshair();
        }
    }

Mendengar dan memproses respon input pemain secara *real-time*:

  • Secara konsisten menjalankan fungsi rotasi pandangan dan translasi posisi berjalan.
  • Input.GetMouseButtonDown(0) || Input.GetKeyDown(KeyCode.E) : Jika mendeteksi tombol Klik Kiri Mouse **atau** tombol 'E' pada keyboard ditekan, skrip akan langsung memicu fungsi pengecekan interaksi objek di arah pandangan mata.

5. Mekanisme Rotasi Kamera First Person (HandleRotation)

    void HandleRotation()
    {
        float mouseX = Input.GetAxis("Mouse X") * mouseSensitivity;
        float mouseY = Input.GetAxis("Mouse Y") * mouseSensitivity;

        rotationX -= mouseY;
        rotationX = Mathf.Clamp(rotationX, upperLookLimit, lowerLookLimit);

        playerCamera.transform.localRotation = Quaternion.Euler(rotationX, 0f, 0f);
        transform.Rotate(Vector3.up * mouseX);
    }

Memisahkan kalkulasi sumbu rotasi horizontal dan vertikal:

  • Mathf.Clamp(rotationX, upperLookLimit, lowerLookLimit); : Membatasi nilai derajat rotasi vertikal agar kamera berhenti berputar saat melihat tegak lurus ke langit atau ke tanah.
  • Quaternion.Euler(rotationX, 0f, 0f); : Menerapkan rotasi dongakan ke atas-bawah hanya pada komponen internal **Kamera** saja.
  • transform.Rotate(Vector3.up * mouseX); : Memutar seluruh **Badan Kapsul Karakter** ke arah kiri-kanan sesuai gerak horizontal mouse.

6. Logika Pergerakan Fisika Karakter (HandleMovement)

    void HandleMovement()
    {
        float moveX = Input.GetAxis("Horizontal");
        float moveZ = Input.GetAxis("Vertical");

        Vector3 moveDirection = transform.forward * moveZ + transform.right * moveX;

        if (moveDirection.magnitude > 1f) moveDirection.Normalize();

        characterController.Move(moveDirection * moveSpeed * Time.deltaTime);
    }

Menerjemahkan input tombol WASD menjadi perpindahan koordinat 3D:

  • transform.forward * moveZ + transform.right * moveX; : Memastikan pergerakan maju-mundur atau menyamping selalu menyesuaikan dengan arah hadap orientasi lokal tubuh karakter saat itu.
  • moveDirection.Normalize(); : Menormalkan panjang vektor gerak agar kecepatan jalan diagonal (saat menekan tombol W dan A bersamaan) tidak menjadi lebih cepat daripada kecepatan jalan lurus biasa.
  • characterController.Move(...) : Memindahkan posisi karakter menggunakan sistem tabrakan CharacterController agar pemain tidak menembus dinding bangunan map.

7. Alur Prioritas Target Interaksi (TryInteractFromCrosshair)

    void TryInteractFromCrosshair()
    {
        Vector2 screenCenter = new Vector2(Screen.width / 2f, Screen.height / 2f);
        if (TryClickUI(screenCenter)) return;
        TryClickWorldObject();
    }
  • Menghitung titik tengah layar berdasarkan resolusi monitor yang digunakan (Screen.width / 2f).
  • Sistem menerapkan urutan prioritas: Pertama-tama mencoba berinteraksi dengan elemen UI Canvas terlebih dahulu (TryClickUI). Jika di titik tengah tersebut tidak ada komponen UI, barulah beralih ke metode kedua untuk mendeteksi komponen objek fisik 3D dunia game (TryClickWorldObject).

8. Deteksi Jarak Klik Kombol UI (TryClickUI)

    bool TryClickUI(Vector2 screenPosition)
    {
        PointerEventData pointerData = new PointerEventData(EventSystem.current) { position = screenPosition };
        List<RaycastResult> results = new List<RaycastResult>();
        EventSystem.current.RaycastAll(pointerData, results);

        foreach (RaycastResult result in results) {
            Button button = result.gameObject.GetComponentInParent<Button>();
            if (button == null || !button.interactable) continue;

            float distanceToButton = Vector3.Distance(playerCamera.transform.position, button.transform.position);
            if (distanceToButton > interactDistance) continue;

            ExecuteEvents.Execute(button.gameObject, pointerData, ExecuteEvents.pointerClickHandler);
            return true;
        }
        return false;
    }

Mekanisme penembakan sinar (*raycast*) khusus untuk antarmuka tombol UI:

  • EventSystem.current.RaycastAll(...) : Menembakkan laser dari tengah layar monitor untuk mendeteksi apakah laser tersebut menembus komponen tombol UI game di Canvas.
  • Vector3.Distance(...) : Menghitung jarak matematis antara posisi mata kamera pemain dengan posisi tombol UI tersebut di dalam scene ruang 3D.
  • if (distanceToButton > interactDistance) continue; : Jika posisi tombol UI berada di tempat yang terlampau jauh (melebihi batas jangkauan), maka perintah klik akan diabaikan demi menjaga aspek realisme.
  • ExecuteEvents.Execute(...) : Jika lolos validasi jarak, skrip akan mensimulasikan dan memicu fungsi klik (pointerClickHandler) pada tombol tersebut secara paksa lewat program.

9. Deteksi Tabrakan Fisika Objek 3D (TryClickWorldObject)

    void TryClickWorldObject()
    {
        Ray ray = playerCamera.ViewportPointToRay(new Vector3(0.5f, 0.5f, 0f));
        if (Physics.Raycast(ray, out RaycastHit hit, interactDistance)) {
            if (hit.collider.gameObject == gameObject) return;

            Button button = hit.collider.GetComponentInChildren<Button>();
            if (button == null) button = hit.collider.GetComponentInParent<Button>();

            if (button != null && button.interactable) {
                button.onClick.Invoke();
            }
        }
    }

Mekanisme penembakan laser fisik untuk mendeteksi komponen 3D collider di dunia game:

  • ViewportPointToRay(new Vector3(0.5f, 0.5f, 0f)); : Menciptakan seutas garis lurus khianat (Ray) yang ditembakkan tepat dari lensa kamera mengarah lurus ke depan luar.
  • Physics.Raycast(ray, out RaycastHit hit, interactDistance) : Menjalankan fungsi tabrakan sinar fisik Unity. Jika sinar tersebut menabrak suatu komponen Collider dalam batas jarak tertentu, data objek yang tertabrak dikunci ke dalam variabel hit.
  • GetComponentInParent
  • button.onClick.Invoke(); : Jika komponen tombol interaksi ditemukan pada objek fisik tersebut, sistem akan langsung memanggil fungsi Invoke() untuk mengeksekusi fungsi tujuan (seperti memicu alarm, membuka pintu, atau mengamankan ID Card) secara instan.
