Analisis Detail Skrip Interaksi Kartu Identitas (UTS_IDCardScript.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip UTS_IDCardScript.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman logika interaksi tombol UI, efek visual rotasi objek, serta hubungannya dengan skrip pengontrol utama game.

1. Library/Namespace Imports

using UnityEngine;
using UnityEngine.UI;

Mengimpor pustaka (library) bawaan Unity yang diperlukan oleh skrip ini:

  • using UnityEngine; : Library dasar wajib Unity untuk mengenali komponen-komponen dasar seperti MonoBehaviour, GameObject, Time, dan fungsi bawaan engine lainnya[cite: 3].
  • using UnityEngine.UI; : Digunakan agar skrip mengenali dan dapat mengendalikan komponen UI interaktif klasik bawaan Unity, yaitu Button[cite: 3].

2. Deklarasi Kelas & Variabel Komponen (Serialized Fields)

public class UTS_IDCardScript : MonoBehaviour
{
    [SerializeField] GameObject _idCard;
    [SerializeField] Button _idCardButtonA;
    [SerializeField] Button _idCardButtonB;
    [SerializeField] UTS_GameController _gameController;
  • public class UTS_IDCardScript : MonoBehaviour : Deklarasi kelas bernama UTS_IDCardScript yang diturunkan dari MonoBehaviour agar skrip ini bisa ditempelkan sebagai komponen pada objek di Unity[cite: 3].
  • Atribut [SerializeField] digunakan untuk memaksa variabel privat di bawah ini agar tetap muncul di jendela Inspector Unity, sehingga kita bisa menghubungkan objeknya secara langsung dengan metode *drag-and-drop*[cite: 3]:
    • _idCard : Menyimpan referensi objek fisik atau visual 3D dari kartu identitas di dalam game[cite: 3].
    • _idCardButtonA dan _idCardButtonB : Menyimpan referensi dua buah komponen UI tombol (Button) yang akan diklik oleh pemain untuk berinteraksi dengan kartu[cite: 3].
    • _gameController : Menyimpan referensi ke skrip master UTS_GameController untuk mengirimkan data perubahan status game[cite: 3].

3. Registrasi Event Klik Tombol (Fungsi Start)

    // Start is called once before the first execution of Update after the MonoBehaviour is created
    void Start()
    {
        _idCardButtonA.onClick.AddListener(OnIDCardClick);
        _idCardButtonB.onClick.AddListener(OnIDCardClick);
    }

Fungsi bawaan Unity yang dijalankan otomatis satu kali di awal permainan[cite: 3]:

  • onClick.AddListener(OnIDCardClick); : Mendaftarkan atau mengikat fungsi OnIDCardClick ke dalam sistem event klik tombol[cite: 3].
  • Artinya, setiap kali pemain mengklik _idCardButtonA ataupun _idCardButtonB pada antarmuka game, Unity akan langsung memanggil dan menjalankan fungsi logika OnIDCardClick secara otomatis[cite: 3].

4. Efek Visual Animasi Berputar (Fungsi Update)

    // Update is called once per frame
    void Update()
    {
        transform.Rotate(0f, 90f * Time.deltaTime, 0f);
    }

Fungsi bawaan Unity yang dieksekusi terus-menerus di setiap frame permainan[cite: 3]:

  • transform.Rotate(0f, 90f * Time.deltaTime, 0f); : Berfungsi untuk memutar objek kartu identitas ini pada sumbu Y (horisontal) sebesar 90 derajat per detik[cite: 3].
  • Time.deltaTime : Sangat penting digunakan sebagai faktor pengali agar kecepatan rotasi objek tetap berjalan konstan dan mulus di berbagai perangkat komputer, terlepas dari perbedaan performa FPS (*Frames Per Second*) masing-masing perangkat[cite: 3].

5. Logika Pengambilan Kartu (Fungsi OnIDCardClick)

    public void OnIDCardClick()
    {
        _gameController.SetHasKeyCard();
        _idCard.SetActive(false);
    }
}

Fungsi publik inti yang akan dijalankan ketika salah satu tombol interaksi kartu diklik oleh pemain[cite: 3]:

  • _gameController.SetHasKeyCard(); : Menghubungi skrip master game controller untuk menjalankan fungsi SetHasKeyCard(), yang akan mengubah status permainan bahwa pemain kini telah berhasil mengantongi kartu kunci[cite: 3].
  • _idCard.SetActive(false); : Menonaktifkan objek kartu identitas di dalam dunia permainan[cite: 3]. Efeknya, kartu tersebut akan langsung menghilang dari layar dan tidak bisa diinteraksi lagi karena dianggap sudah diambil[cite: 3].
