Analisis Detail Skrip Tombol Darurat (UTS_EmergencyButton.cs)

Halaman ini berisi penjelasan lengkap baris demi baris dari skrip UTS_EmergencyButton.cs. Dokumentasi ini dibuat untuk mempermudah pemahaman logika interaksi tombol darurat UI yang digunakan pemain untuk mengaktifkan akses pintu keluar darurat utama.

1. Library/Namespace Imports

using UnityEngine;
using UnityEngine.UI;

Mengimpor pustaka (library) bawaan Unity yang diperlukan oleh skrip ini:

  • using UnityEngine; : Library dasar utama Unity agar skrip mengenali komponen penting seperti MonoBehaviour dan SerializeField.
  • using UnityEngine.UI; : Digunakan agar skrip dapat mengakses dan mengontrol komponen sistem UI klasik Unity, khususnya kelas Button.

2. Deklarasi Kelas & Variabel Referensi Objek

public class UTS_EmergencyButton : MonoBehaviour
{
    [SerializeField] UTS_GameController _gameController;
    [SerializeField] Button _emergencyExitButton;
  • public class UTS_EmergencyButton : MonoBehaviour : Deklarasi kelas utama bernama UTS_EmergencyButton yang mewarisi sifat MonoBehaviour agar dapat diletakkan pada GameObject di dalam Unity.
  • Atribut [SerializeField] memaksa variabel privat agar terekspos di dalam Inspector Unity, memungkinkan pengembang menghubungkan referensi objek menggunakan metode *drag-and-drop*:
    • _gameController : Menyimpan referensi skrip master pengontrol game (UTS_GameController) untuk mengirim sinyal aktivasi.
    • _emergencyExitButton : Menyimpan referensi ke komponen UI Tombol (Button) fisik darurat yang akan ditekan oleh pemain di layar.

3. Inisialisasi Event Listener (Fungsi Start)

    // Start is called once before the first execution of Update after the MonoBehaviour is created
    void Start()
    {
        _emergencyExitButton.onClick.AddListener(OnEmergencyButtonClick);
    }

Fungsi siklus hidup Unity yang dieksekusi otomatis satu kali saat objek pertama kali aktif:

  • _emergencyExitButton.onClick.AddListener(OnEmergencyButtonClick); : Mendaftarkan fungsi lokal OnEmergencyButtonClick ke dalam event klik tombol onClick.
  • Melalui baris ini, setiap kali pemain berinteraksi dan mengklik tombol darurat tersebut di layar, Unity akan langsung memicu dan mengeksekusi fungsi logika OnEmergencyButtonClick.

4. Fungsi Update Default

    // Update is called once per frame
    void Update()
    {

    }
  • Update() : Fungsi bawaan Unity yang berjalan berulang kali di setiap frame permainan. Pada skrip ini fungsi dibiarkan kosong karena sistem tombol darurat murni digerakkan oleh event interaksi klik (*Event-Driven Logics*), sehingga tidak memerlukan pemrosesan berkala setiap frame yang dapat membuang performa CPU game.

5. Eksekusi Logika Klik Tombol (OnEmergencyButtonClick)

    public void OnEmergencyButtonClick()
    {
        _gameController.SetEmergencyExitActive();
    }
}

Fungsi publik yang bertindak sebagai penerima sinyal interaksi klik tombol darurat:

  • _gameController.SetEmergencyExitActive(); : Berfungsi untuk memanggil fungsi SetEmergencyExitActive() yang berada di skrip master pengontrol game.
  • Pemanggilan fungsi ini akan mengubah status game menjadi *Emergency Exit Active*, memicu umpan balik visual kesuksesan, memperbarui daftar centang misi ke-3 (Open Emergency Door), serta memperbarui ikon lokasi pada peta mini.
